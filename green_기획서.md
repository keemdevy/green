## 시장 배경

### 글로벌 검증
- **Tea (미국)**: 2025년 7월 ChatGPT를 제치고 앱스토어 1위, 4백만+ 여성 가입
- **Hinge**: 2025년 AI 추천 엔진으로 매칭 15% 증가
- **Sonder (런던)**: 의도적 마찰 가입 + 무드보드 프로필, 유료 마케팅 0원으로 6,500명 유저 확보

### 한국 시장 진공
- Tea 같은 평판 매칭 앱: **0개**
- 위피·글램·틴더: 모두 남성 중심 알고리즘 (남성이 결제 위해 여성에게 도배하는 구조)
- 한국 데이팅앱 사용자 75.7%: "서비스 앱으로 만난 관계는 진정성 부족"
- 여성 사용자 거부 이유 1순위 = **안전**, 2순위 = **시간 낭비/고스팅**, 3순위 = **외모 평가받는 느낌**

### 핵심 통찰
> 여성은 "데이트할 사람"보다 "안전하게 데이트할 방법"에 훨씬 더 큰 결제 의지가 있다.

## 핵심 컨셉

### 기존 데이팅앱 vs 그린

| 기존 데이팅앱 | 그린 |
|--------------|------|
| 남성이 좋아요 누르고 여성이 받는다 | **여성이 남성을 평가하고, 평가받은 남성만 노출된다** |
| 가입 즉시 매칭 풀 진입 | **남성은 최소 3명의 여성에게 그린플래그 받아야 활성화** |
| 외모 = 매력 | **평판 = 매력** |
| 매칭 = 끝 | **매칭 후에도 평가 가능. 평점 떨어지면 비활성화** |

### 여성 측 인센티브 (왜 가입하나)
1. **무료** — 여성은 100% 무료
2. **평가 권력** — 한국에선 처음 있는 권력 구조
3. **안전** — 남성은 모두 본인인증 + 다른 여성의 검증 통과
4. **시간 절약** — 안전한 남성 풀에서만 고름
5. **커뮤니티** — Tea처럼 다른 여성과 대화·정보 공유 가능

### 남성 측 인센티브 (왜 결제하나)
1. **여성이 많다** — 여성 가입자 압도적이라 자연스레 남성도 따라옴
2. **차별화** — 그린플래그 많이 받으면 = 매력 증명
3. **진지한 매칭** — 검증된 남성만 노출 = 가벼운 매칭 X
4. **결제 = 지위** — 월 19,900원 / 29,900원 / VIP 49,900원 티어

## 작동 방식

### 1단계: 가입 (남성)
- 카카오 + PASS 본인인증 (필수)
- 사진 등록 (셀카 검증 + AI 동일인 확인)
- 자기소개 작성 (200자, AI 검열)
- 가입 직후엔 **"검토 중" 상태** → 매칭 기능 잠금

### 2단계: 평판 획득 (남성의 도전)
- 여성 사용자만 **"검토 풀"** 접근 가능
- 여성은 검토 중인 남성 프로필 보고 **그린플래그** 또는 **패스**
- 그린플래그 = **사전 정의 옵션 1~3개 선택** (자유 텍스트 X)
- **3명 이상 여성에게 그린플래그 받으면 매칭 활성화**

### 3단계: 매칭 (여성 100% 권력)
- 활성화된 남성만 여성의 메인 피드에 노출
- 각 남성 프로필에 그린플래그 받은 횟수 + 평가 코멘트 표시
- 여성이 좋아요 → 매칭 성사 → 채팅 시작
- **남성은 절대 먼저 좋아요 못 누름**
- 24시간 매칭 만료 (Bumble 스타일)

### 4단계: 매칭 후 평가
- 만남 후 양쪽 익명 평가 가능
- 여성이 남성에게 **레드플래그** 주면 → 그 남성 매칭 풀에서 자동 강등
- 3회 레드플래그 시 → 자동 비활성화 + 환불 X
- **남성은 여성에게 평가권 없음** (이게 핵심 광기)
- 이의 신청권 보장

### 그린플래그 사전 정의 옵션 (20개)
자유 텍스트 절대 금지. 명예훼손 차단의 핵심.

긍정 옵션 예시:
- 말투가 정중함
- 프로필 진솔함
- 사진이 자연스러움
- 자기소개가 구체적
- 직업 정보 신뢰감
- 취미가 흥미로움
- 시간 관리 잘할 듯
- 건강한 라이프스타일
- 지적 호기심 보임
- 가족 가치관 좋음
- (총 20개)

## 기술 스택

### 환경 결정

| 영역 | 선택 | 이유 |
|------|------|------|
| 프론트엔드 | **React Native + Expo** | iOS/Android 한 번에, 빠름 |
| 백엔드 | **Supabase** | DB + 인증 + 실시간 + 스토리지 통합, 무료티어 큼 |
| 본인인증 | **PASS (KISA 인증)** | 한국 필수, 봇 차단 핵심 |
| 결제 | **토스페이먼츠** + 인앱결제 우회 (웹뷰) | 수수료 절감 |
| AI 검열 | **OpenAI Moderation API** + Claude API | 텍스트 자동 필터링 |
| 이미지 분석 | **AWS Rekognition** 또는 Vercel Image | 사진 진위 확인 |
| 푸시알림 | **Firebase Cloud Messaging** | 무료, 안정 |
| 분석 | **PostHog** | 무료티어, 사용자 행동 추적 |

### 프로젝트 구조

```
green-app/
  ├── app/              # Expo Router
  │   ├── (auth)/       # 로그인·인증
  │   ├── (tabs)/       # 메인 탭
  │   │   ├── review/   # 검토 풀 (여성)
  │   │   ├── matches/  # 매칭 (여성·활성 남성)
  │   │   ├── chats/    # 채팅
  │   │   └── profile/  # 프로필
  │   └── onboarding/
  ├── lib/
  │   ├── supabase.ts
  │   ├── ai/           # Claude API 검열
  │   └── pass/         # PASS 본인인증
  └── components/
```

### Supabase DB 스키마 (핵심 테이블)

```sql
-- 사용자
users (
  id uuid primary key,
  gender text check (gender in ('M', 'F')),
  pass_verified boolean default false,
  status text default 'pending', -- pending, active, banned
  green_flags_count int default 0,
  created_at timestamp
);

-- 프로필
profiles (
  user_id uuid references users,
  photos text[],          -- S3 URL 배열
  bio text,               -- AI 검열 통과만
  interests text[]        -- 5개 태그
);

-- 그린플래그 (여성 → 남성)
green_flags (
  id uuid primary key,
  from_user_id uuid,      -- 여성
  to_user_id uuid,        -- 남성
  reasons text[],         -- 사전 정의 옵션 1~3개
  created_at timestamp
);

-- 매칭
matches (
  id uuid primary key,
  user_a uuid,
  user_b uuid,
  status text,            -- pending, active, expired
  expires_at timestamp    -- 24시간 후
);

-- 평가 (만남 후)
evaluations (
  match_id uuid,
  from_user_id uuid,
  flag_type text,         -- green, red
  reasons text[],
  created_at timestamp
);
```

**핵심 KPI: 7일 안에 여성 사전 가입 200명**

#### 매일 측정
- 랜딩페이지 방문자
- 베타 신청 완료
- 신청 → 인터뷰 동의 비율
- 인스타 광고 CPC, CTR, CPM

#### 7일 차 결정 매트릭스

| 가입자 수 | 의미 | 대응 |
|----------|------|------|
| 50명 미만 | 메시징 안 통함 | 카피·소재·타깃 다 갈아엎기 |
| 50~150명 | 컨셉은 OK, 마케팅 부족 | 광고 예산 50% 증액 + 인플루언서 푸시 |
| 150~300명 | 컨셉·메시징 둘 다 OK | 그대로 진행, MVP 개발 시작 |
| 300명+ | 폭발적 반응 | 즉시 MVP 가속 + 시드 IR 준비 |

#### 30분 인터뷰 질문 5개
1. 마지막으로 데이팅앱 쓴 게 언제고 왜 그만뒀나요?
2. 그린이 어떤 점이 가장 끌렸나요?
3. 남성 검증한다고 하면, 어떤 정보가 가장 알고 싶나요?
4. 월 얼마면 결제하시겠어요? (남성 사용자 가정)
5. 친구에게 추천한다면 어떤 한 줄로 설명하시겠어요?

→ 5번 답변이 그대로 마케팅 카피가 됨

- 첫 매칭까지 시간 (여성 가입 후 7일 이내 첫 매칭)

## 리스크와 대응

### 1. 여성이 안 모이면 = 게임 끝
- **대응:** 사전 검증 14일 안에 200명 못 모으면 컨셉 수정
- 변호사·디자인 비용 매몰. 빨리 인지하는 게 핵심

- **대응:** 메시징을 "남성에게 불리"가 아닌 **"진지한 남성을 보호"** 로 포지셔닝
- "여성에게 검증받은 남성 = 결혼 시장에서 신뢰받는 남성"
- 남성 자기개발 코치 기능 강조

### 3. 명예훼손 소송 1건이라도 발생 → 평판 즉사
- **대응:**
  - 자유 텍스트 절대 금지
  - 사전 정의 옵션만 사용
  - AI 자동 검열
  - 이의 신청권
  - 신고 24시간 처리

### 4. 데이터 유출 → Tea처럼 무너짐

  - 이미지 워터마크 의무
  - 스크린샷 자동 차단

### 5. 여성 인플루언서 1명이 부정적 후기 → 가입 폭발 ↓
